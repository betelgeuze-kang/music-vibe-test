듣기 기록은 이 브라우저에 남습니다.
익명 분석은 따로 허락한 뒤에만 나갑니다.

계정 없이도 모든 핵심 기능을 쓸 수 있습니다. 프로필, 지난 기록, 곡 반응, 방문 날짜, Weekly Vibe는 이 브라우저에 보관됩니다. 익명 서비스 분석과 광고 관련 동의는 서로 다른 선택이며, 분석을 허용해도 광고 저장이나 맞춤형 광고가 켜지지 않습니다.

시행·마지막 검토: 2026-07-13 · Privacy release PV1

1. 브라우저 안에 보관하는 기록

  • 열 번의 선택으로 만든 여섯 방향과 익명 프로필 ID
  • 프로필 타임라인, 최근 선곡, 곡에 남긴 표시, Weekly Vibe
  • 언어, 개인정보 선택, 재방문을 알아보기 위한 방문 날짜

이 정보는 보통 localStorage에 남습니다. 데이터 설정 화면에서 전체 JSON을 내려받거나, 이전 파일을 가져오거나, 종류별로 비우거나, 모두 지울 수 있습니다. 이름·이메일·전화번호·회원 계정은 묻지 않습니다.

2. 친구에게 건네는 링크

친구 비교 링크에는 이름 대신 여섯 개의 취향 값과 익명 ID가 들어갑니다. 값은 URL의 # 뒤에 놓이며, 일반적인 HTTP 요청에서는 서버나 CDN으로 전송되지 않습니다.

3. 서로 나누어 놓은 세 가지 선택

필수 기능

브라우저 안의 기록은 테스트·추천·타임라인·주간 기록을 위해 사용됩니다. 익명 분석을 거절해도 핵심 기능은 그대로 쓸 수 있습니다.

익명 서비스 분석

허용한 경우에만 화면 이동, 테스트 완료, 추천 생성, 외부 음악 링크 열기, 곡 반응, Weekly Vibe 열기 같은 흐름이 GA4로 전송될 수 있습니다. 이때에만 영구 익명 방문자 ID를 만듭니다. 동의를 철회하면 분석 저장을 다시 denied로 바꾸고, 브라우저에 남은 분석 ID와 대기 이벤트를 지웁니다.

광고 측정과 맞춤형 광고

분석과 별개의 선택입니다. 현재 광고 제공이 꺼져 있어 선택 항목도 비활성 상태입니다. ‘익명 분석 허용’은 ad_storage, ad_user_data, ad_personalization을 허용하지 않습니다.

4. 외부 음악 서비스

Spotify·YouTube·Apple Music 링크를 열면 이 사이트를 벗어나 해당 서비스의 개인정보·쿠키 정책이 적용됩니다. My Music Vibe는 그곳에서 실제로 무엇을 재생했는지 읽지 않습니다.

5. 현재 광고 상태

광고 제공 꺼짐

공개 ads.txt에는 Google AdSense 판매자 레코드 pub-1386368370627622가 등록돼 있습니다. 이것은 이 도메인에 승인된 계정을 알리는 파일일 뿐 광고를 불러오는 코드가 아닙니다. 현재 릴리스에는 AdSense 제공 스크립트, 활성 광고 슬롯, 광고 쿠키, 맞춤형 광고 흐름이 없습니다.

광고 실험을 시작하려면 실제 처리 목적, 동의 모드, 거부 방법, 배치 위치, 시행일을 이 정책에 다시 적어야 합니다. 질문·오디오 카드·내비게이션·모달·추천곡 카드·플랫폼 링크·곡 반응·오류·빈 상태에는 광고를 두지 않습니다.

6. 테스트 오디오

짧은 테스트 소리 여덟 개는 외부 녹음과 샘플 없이 브라우저에서 직접 합성합니다. 소스·권리 manifest·라이선스는 오디오 권리 페이지에 공개합니다.

7. 선택을 다시 열거나 기록을 지우는 법

앱 푸터의 ‘개인정보 설정’에서 언제든 동의 화면을 다시 열 수 있습니다. ‘내 데이터와 개인정보’ 화면에서는 전체 기록 내보내기·가져오기, 항목별 삭제, 분석 ID 삭제, 전체 삭제를 할 수 있습니다.

8. 문의와 정정

권리·개인정보·링크·문구 정정은 GitHub 이슈로 받을 수 있습니다. 광고나 데이터 처리 방식이 실질적으로 바뀌면 새 시행일과 함께 이 페이지를 고칩니다.
