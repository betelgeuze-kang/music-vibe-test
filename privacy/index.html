PRIVACY & MONETIZATION / CR1

취향 기록은 브라우저에,
광고는 아직 꺼진 상태입니다.

현재 My Music Vibe는 회원가입 없이 동작하며, 프로필·최근 선곡·곡 반응·주간 기록을 사용자의 브라우저 저장소에 보관합니다. CR1에서는 향후 광고 도입에 필요한 경계를 미리 정했지만 실제 광고 코드와 광고 쿠키는 활성화하지 않았습니다.

시행·마지막 검토: 2026-07-12

1. 현재 수집·저장하는 정보

브라우저 안에만 저장되는 정보

  • 10개 선택으로 생성된 6개 취향 값과 익명 프로필 ID
  • 프로필 타임라인, 최근 선곡, 곡 반응, Weekly Vibe
  • 언어·필수/선택 분석 동의 상태와 방문 날짜

이 데이터는 기본적으로 localStorage에 저장되며, 사용자가 브라우저 데이터를 지우거나 서비스의 삭제 기능을 사용하면 제거됩니다. 별도 계정·이름·이메일·전화번호를 요구하지 않습니다.

친구 비교 링크

친구 비교 값은 URL의 # 뒤에 들어갑니다. 일반적인 HTTP 요청에서 이 fragment는 서버나 CDN으로 전송되지 않습니다. 링크에는 이름이나 연락처 대신 익명 ID와 6개 취향 값이 포함됩니다.

2. 선택적 분석

서비스 개선을 위한 선택적 분석은 사용자가 동의한 경우에만 전송합니다. 동의하지 않아도 취향 테스트·추천·친구 비교 기능을 사용할 수 있습니다.

  • 예: 화면 이동, 테스트 완료, 추천 생성, 플랫폼 링크 클릭, 곡 반응, Weekly Vibe 확인
  • 전송하지 않는 정보: 이름, 이메일, 친구 이름, 외부 스트리밍 계정, Spotify·YouTube·Apple의 실제 청취 이력

3. 외부 음악 서비스

추천곡의 Spotify·YouTube·Apple Music 링크를 열면 각 사업자의 개인정보·쿠키 정책이 적용됩니다. My Music Vibe는 외부 서비스에서 사용자가 실제로 무엇을 듣는지 수집하지 않습니다.

상업 음원의 제목·아티스트와 자체 작성 설명만 표시하며, 음원 파일·가사·앨범 커버를 이 서버에 복제하지 않습니다.

4. 광고 상태와 도입 원칙

현재 광고 비활성

현재 페이지에는 AdSense를 포함한 제3자 광고 스크립트, 광고 슬롯, 광고 쿠키가 없습니다. 소스 코드의 광고 정책도 기본 OFF이며 실제 publisher ID가 비어 있으면 광고를 렌더링할 수 없습니다.

향후 광고를 활성화하기 전

  • 실제 광고 사업자·publisher ID와 ads.txt를 확인합니다.
  • 이 정책에 광고 사업자, 쿠키·기기 식별자, 목적, 보유·거부 방법을 추가합니다.
  • 법령과 서비스 지역에 따라 필요한 동의 관리 플랫폼(CMP)을 적용합니다.
  • 맞춤형 광고는 유효한 동의가 확인되기 전에는 활성화하지 않습니다.
  • 온보딩·오디오 선택·내비게이션·모달·피드백 버튼 주변에는 광고를 배치하지 않습니다.

광고가 활성화되는 경우의 일반 고지

제3자 광고 사업자는 광고 제공·빈도 제한·성과 측정을 위해 쿠키 또는 유사 기술을 사용할 수 있습니다. Google 광고를 사용하는 경우 Google과 파트너가 이 사이트 또는 다른 사이트 방문 기록을 바탕으로 광고를 제공할 수 있고, 사용자는 Google 광고 설정 등 제공되는 방법으로 맞춤형 광고를 관리하거나 거부할 수 있습니다. 이 문구는 현재 광고가 활성화됐다는 뜻이 아니라, 활성화 전에 구체적인 사업자와 선택권을 다시 고지하기 위한 준비 항목입니다.

5. 광고 배치 안전선

향후 광고를 도입하더라도 다음 위치에는 표시하지 않습니다.

  • 10개 취향 질문과 A/B 오디오 카드 내부
  • 재생·선택·다음·뒤로가기·공유·삭제·곡 반응 버튼 인접 영역
  • 상단·모바일 내비게이션과 개인정보·확인 대화상자
  • 오류·빈 상태·친구 초대 입력 화면
  • 콘텐츠로 오인될 수 있는 곡 추천 목록 내부

허용 후보도 충분한 본문 뒤의 독립 구획으로 제한하며, 항상 `광고` 또는 `Advertisement`로 표시하고 콘텐츠와 시각적으로 구분합니다.

6. 오디오 권리

취향 테스트 미리듣기는 제3자 음원을 사용하지 않고 브라우저에서 직접 합성합니다. 제작 방식·라이선스·권리 manifest는 오디오 출처와 권리 페이지에서 확인할 수 있습니다.

7. 문의와 정책 변경

권리·개인정보·링크 수정 요청은 GitHub 이슈로 접수할 수 있습니다. 광고 사업자나 처리 방식이 바뀌면 시행일과 변경 내용을 이 페이지에 반영합니다.
