WHAT STAYS / WHAT MAY LEAVE

The listening note stays in this browser.
Analytics leaves only after a separate choice.

The product works without an account. Profiles, earlier pages, feedback, visits, and Weekly Vibe are stored locally. Optional product analytics and advertising permissions are separate decisions. Allowing analytics does not allow advertising storage or personalized ads.

Effective and last reviewed July 13, 2026 · Privacy release PV1

1. Information kept in this browser

  • The six-direction profile and anonymous profile ID created from ten choices
  • Earlier profile pages, recent selections, track feedback, and Weekly Vibe
  • Language, privacy preferences, and dates used for the return loop

These records normally live in localStorage. They can be exported, cleared by category, or deleted altogether from the Data & Privacy page. No name, email address, telephone number, or account is required.

2. A friend link

A comparison link places six taste values and an anonymous ID after the # in the URL. The fragment is not sent in an ordinary HTTP request to the server or CDN. The link contains no name or contact detail.

3. Separate privacy choices

Essential functions

Local storage needed for the notebook can be used without optional analytics. Refusing analytics does not remove the core test, recommendations, Weekly Vibe, or friend comparison.

Anonymous product analytics

If allowed, events such as route changes, test completion, recommendation creation, external link clicks, feedback, and Weekly Vibe views may be sent to GA4. Only then is a persistent anonymous visitor ID created. Revoking analytics removes that local analytics ID, clears queued events, and returns Google consent for analytics storage to denied.

Advertising measurement and personalized advertising

These remain separate and denied. Advertising delivery is currently off, so the controls are disabled. Enabling anonymous analytics never grants ad_storage, ad_user_data, or ad_personalization.

4. External music services

Opening Spotify, YouTube, or Apple Music leaves this site and brings the user under that service’s own privacy and cookie terms. My Music Vibe does not read what the user actually plays there.

5. Current advertising status

Advertising delivery is off

The public ads.txt contains the authorized Google AdSense seller record for pub-1386368370627622. A seller record identifies the account authorized for the domain; it does not load an ad. The current release contains no AdSense delivery script, active ad slot, advertising cookie, or personalized-ad flow.

Before any experiment begins, this policy must name the actual processing, consent mode, opt-out choices, placement, and effective date. Ads may not sit inside onboarding, audio choices, navigation, dialogs, track cards, platform links, feedback controls, errors, or empty states.

6. The listening-test audio

The eight short test sounds are generated in the browser without third-party recordings or samples. Their source, rights record, and license are published on the audio-rights page.

7. Changing a choice or deleting data

Use “Privacy choices” in the application footer to reopen the consent panel. The Data & Privacy page can export all local records, import a previous export, remove a category, delete the analytics identity, or clear everything stored by the application.

8. Questions and corrections

Rights, privacy, and link corrections may be reported through a GitHub issue. A material change to advertising or data processing will be reflected here with a new effective date.
